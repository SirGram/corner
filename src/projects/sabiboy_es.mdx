## ¿Por qué Rust?

---

Bueno, había oído hablar de Rust desde hace tiempo y quería probarlo. Vi a alguien programando su propio emulador de NES en este lenguaje y pensé "Pues no tá mal".
Parece que la programación de emuladores y de sistemas de bajo nivel son excelentes aplicaciones por cómo maneja la memoria.

<br />
## El Camino de la Emulación
---
El objetivo principal era crear un emulador funcional para la Gameboy DMG original. Antes de sumergirme en ello, estudié sistemas más sencillos como el chip-8 para entender los fundamentos antes de abordar una arquitectura más compleja.

## La Biblioteca

---

Quería hacer una biblioteca server-side que me permitiera acceder fácilmente al emulador desde el navegador.

Para poder usar el emulador, utilicé wasm-bindgen para compilar el código de Rust a un archivo wasm.

Algunas de las características que acabé añadiendo:

-Una biblioteca de juegos donde puedes añadir y gestionar ROMs

-Un sistema de savestates de estado que usa MongoDB

-Ajustes personalizables para el emulador como diferentes paletas de colores o debug

-Modo offline para usuarios sin usuario
